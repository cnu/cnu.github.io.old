Moved to Octopress

Sep 28th, 2013

I have decided to move my site/blog to Octopress from the old nanoc based one. Both are static website generators, but since octopress is based on the much popular jekyll/Github based platform, I decided to try this out.

The main reason I chose nanoc over jekyll was the ability to have static home “page” as opposed to the blog as the main page and it was easier to begin with. Now that I got (re)introduced to octopress, I feel things are as easy as nanoc. Of course I had some trouble getting the right version of Ruby on Mac OS X, but atlast found a good step-by-step tutorial. For the theme, I found a nice looking one by Justin Kelly and just used it without any changes.

I just had to copy-paste the content from the old site to the new one as it was all mostly written using Markdown. However, I want to fix the projects page a bit – adding more information, making it more structured and easier to parse (by a human).

Right now I think this does a good enough job of running my personal website. Now the only remaining choice is to whether host it on Github or on own personal host.
